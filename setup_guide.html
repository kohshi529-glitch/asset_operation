手法カード＆プロジェクト 設定要領

5大手法スコアリング運用のセットアップ手順（間違い防止版）
📊 スコアリング基準🛠 設定要領📋 候補ボード🖥 システム
考え方はシンプルです。採点基準は1か所（master.jsonのscoring_spec）にだけ置き、5つのプロジェクトには「薄いカード」を貼る。 カードは基準を参照するだけなので、精度を直すときはmaster.jsonの1か所を編集するだけで全手法に効きます。 以下の順番どおりに進めれば迷いません。

いちばん大事な原則（ここだけは間違えない）

⚠ カードは必ず「指示（カスタム指示）」欄に貼る

プロジェクトには 説明 欄と 指示を追加 欄があります。 カード本文は必ず 指示を追加 欄へ。説明 欄はメモ扱いで、AIの動作には反映されません。 以前うまく効かなかったのは、ここに貼っていたのが原因です。

セットアップ手順

1基準マスターを保管場所に置く

scoring_spec.json の中身を、Google Drive上の master.json に scoring_spec ブロックとして統合します。 これが全手法が読む「唯一の正」。5つのセッションは互いのチャットを読めないので、この共有ファイルだけが手法をつなぐ橋になります。

25つのプロジェクトを用意する

手法ごとに1プロジェクト。すでにある曜日別セッション（月〜金）をそのまま使ってもOK。 下の対応表のとおり、1手法＝1カードで割り当てます。

3対応するカードを「指示を追加」欄に貼る

method_cards.md の中から該当する手法の「```」で囲まれたブロックだけをコピーし、 そのプロジェクトの 指示を追加 欄に貼り付けて保存します。共通テンプレートの見出しごとコピーする必要はありません。1手法につき1ブロックです。

4効いているか動作テストする

保存後、そのプロジェクトで新規チャットを開き、下の一文を送って確認します。指示が読めていれば内容を要約して返します。

この指示で参照する採点基準のキーと、担当手法・満点・判定閾値を答えて。実際の分析はまだしなくていい。

期待する答えの例（決算モメンタムなら）：「scoring_spec.methods["kenmo_momentum"]を参照。7項目21点満点。18〜21で🚀確変=BUY_NOW…」と返ってくれば設定成功です。的外れな返答なら、貼る欄を間違えている可能性が高いので手順3を見直してください。

やってはいけない3つ（アンチパターン）

✗ カードを「説明」欄に貼る
→ 必ず 指示を追加 欄に貼る。説明欄はAIの動作に反映されない。
✗ カードの中に採点基準（点数表）を丸ごと書き込む
→ 基準はmaster.jsonのscoring_specにだけ置く。カードは「参照するだけ」。両方に書くと食い違いの元になり、直すときに二度手間。
✗ 精度を直したいときにカード側を編集する
→ 閾値変更・項目追加・基準の微調整は master.jsonのscoring_specを1か所だけ 編集し、閲覧用HTMLを再生成する。5枚のカードは触らない。

手法の役割・出力形式を変えたい該当カード（指示欄）1枚だけ
その日の地合いで新規を絞りたいmaster.json の market.latest / method_guidance
